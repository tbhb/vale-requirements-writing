# Sample Requirements for Testing

This file contains sample requirements that should trigger the INCOSE GtWR Vale styles.

## R2 - Active Voice

The system was designed to handle user input.
Data is processed by the module.

## R5 - Definite Articles

A system shall provide authentication.
A user must be able to log in.

## R7 - Vague Terms

The system shall provide appropriate responses.
Processing shall be efficient and reasonable.
The software shall respond in a timely manner.

## R8 - Escape Clauses

The system shall operate as much as possible.
Features shall be available if necessary.

## R9 - Open-Ended Clauses

The system shall support formats including but not limited to PDF, etc.

## R10 - Superfluous Infinitives

The system shall be able to process requests.
The module shall be designed to handle errors.

## R15 - Logical Expressions

The system shall support read and/or write operations.

## R16 - Use of Not

The system shall not fail during operation.
Users cannot access restricted areas.

## R17 - Oblique Symbol

The system shall support input/output operations.
Enable/disable features based on configuration.

## R19 - Combinators

The system shall authenticate and authorize users.
The module shall process or reject invalid input.

## R20 - Purpose Phrases

The system shall log events in order to support debugging.
The module exists for the purpose of validation.

## R21 - Parentheses

The system shall store data (temporarily) in memory.

## R24 - Pronouns

It shall respond within 5 seconds.
They must complete registration first.
We recommend using secure protocols.

## R26 - Absolutes

The system shall always be available.
The software shall never crash.
All users shall have access.

## R30 - Repetition

The the system shall process data.

## R31 - Solution Free

The system shall use a database to store records.
The software architecture shall include microservices.

## R32 - Universal Quantification

All users shall authenticate.
Any request shall be logged.
Both systems shall synchronize.

## R35 - Temporal Dependencies

The system shall eventually complete processing.
Data shall be stored before transmission.

## R36 - Consistent Terms

The user shall authenticate.
Users shall receive notifications.

## R37 - Acronyms

The API shall support REST operations.
XML parsing is required.

## R38 - Abbreviations

The system shall process approx 1000 requests.
Max throughput shall be 10 req/sec.
